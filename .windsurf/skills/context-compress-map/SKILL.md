# Context Compress Map

## ワークフロー

1. コンテキストから核心の目標と最小限の裏付け事実を抽出する。
2. 物語的な説明、反復、意思決定に無関係な内容を除去する。
3. 構造化マップ（目的 / 前提 / 主要事実 / 決定済み / 未決 / リスク / 次の手）で出力する。
4. 圧縮により失われる情報にはポインタ（ファイルパス、セクション名等）を残す。
