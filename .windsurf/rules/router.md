# Router Rule

Use this routing dictionary:

- `最新仕様`, `release`, `API変更`, `dependency update` -> `tech-researcher`
- `調査`, `影響範囲`, `参照`, `構造` -> `codebase-explorer`
- `review`, `PR`, `回帰`, `品質` -> `task-reviewer`
- `lint`, `format`, `schema`, `CI`, `test` -> `lint-fix`
- `debug`, `error`, `再現`, `stacktrace` -> `debug-strategist`
- `計画`, `分解`, `implementation steps` -> `task-planner`
- `UI`, `CSS`, `layout`, `component` -> `ui-standardizer`
- `commit`, `branch`, `changelog`, `release` -> `git-helper`
- `agent`, `skill`, `rule`, `workflow`, `template` -> `agent-factory`

If multiple match, prefer the first applicable specialist and keep scope minimal.
