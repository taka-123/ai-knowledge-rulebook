# Research Protocol Skill

技術情報の調査・書き込みにおける厳格なプロトコルを強制するスキルです。

## グローバル規律との関係

グローバルエージェント `tech-researcher` は「調査と接地」を司る。本スキルは、その調査結果がこのリポジトリに書き込まれる際に、正確さと出典の完全性を強制するゲーティングです。

## 情報源の優先順位

```
1. context7（フレームワーク・ライブラリ・SDK）
   ↓
2. 公式ドキュメント（RFC 等を含む）
   ↓
3. 公式 GitHub リポジトリ（Issues, Releases, Discussions）
   ↓
4. WebSearch（公式発表・リリースノート）
   ↓
5. 信頼できるコミュニティ（最後の手段のみ）
```

上位ソースと下位ソースの記述が矛盾する場合は、**上位ソースを採用**し、矛盾事項を明記する。

## 必須プロトコル

### 1. 出典付与

すべての技術的記述には以下が必須：

```markdown
**出典**: [ドキュメント名](URL)
**取得日**: YYYY-MM-DD
```

バージョン情報がある場合は付与する。

### 2. 不確実性の明示

確信度が80%未満の記述には必ず注記を付与する：

```markdown
⚠️ **未確認**: [内容]（公式ドキュメントに明記なし。根拠: [根拠]）
```

「推定」「未確認」「公式に明記なし」のいずれかを明示する。

### 3. クロスチェック

重要な仕様は**複数ソース**で確認し、結果を記録する：

```markdown
- [仕様名]: ソースA・ソースBで一致 → 採用
- [仕様名]: ソースA・ソースBで矛盾 → 最新のソース[X]を採用（理由: [理由]）
```

### 4. 引用の正確性

- キー名・パラメータ名・コマンド構文は**一字一句正確**に記述する。
- 「理解して要約」ではなく「確認してそのまま引用」のスタンスを維持する。
- 推測で埋めることは禁止。不明の場合は未確認として記述する。

## NG パターン

| パターン                                   | 理由                       |
| ------------------------------------------ | -------------------------- |
| 出典なしで技術仕様を記載                   | 検証불可になる             |
| 古いバージョンの情報を現在の仕様として記載 | バージョン混在は誤解の原因 |
| 不確実性を隠蔽して「確定事項」として記述   | 誤情報の拡散リスク         |
| コミュニティ情報のみに依存                 | 信頼度が低い               |

## 審査フロー

本スキルは、技術調査の結果がリポジトリに書き込まれる直前に適用される：

1. 書き込む内容の技術的記述を特定する。
2. 各記述に出典・取得日があるか確認する。
3. 不確実性の注記が必要な箇所を特定し、付与されているか確認する。
4. NG パターンに該当する記述がないか確認する。
5. 不備があれば修正を促し、全項目パスの場合のみ書き込みを続行する。
