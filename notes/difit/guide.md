## 1. 【最強の起動コマンドとエイリアス】

実務で最も頻出するパターンを網羅したエイリアス設定です。オプションの記述順序を統一し、比較対象（ターゲット）の後に各種フラグ（`--clean`等）が続くように整理しています。

`main`や`master`といったデフォルトブランチ名の違いを吸収するため、Gitの標準仕様である `origin/HEAD` を使用するのがベストプラクティスです。

以下のコードブロックを `.zshrc` や `.bashrc` に追記してください。

Bash

```
# 1. 作業中の全変更をレビュー（未追跡ファイル含む）
alias dif='npx difit . --clean --keep-alive --include-untracked'

# 2. リモートのデフォルトブランチ（main/master）と現在の作業状態の差分をレビュー
# ※ origin/HEAD を使うことでブランチ名の揺れを自動吸収します
alias difm='npx difit origin/HEAD . --clean --keep-alive'

# 3. 過去のコミット履歴からfzfで2点間を選択してレビュー（AI生成コードの過去比較用）
alias diffzf='git log --oneline --decorate -100 | fzf | awk "{print $1}" | xargs -I {} npx difit {}~1 {} --clean --keep-alive'
```

> **⚠️ 注意点: 各オプションの役割とエッジケース**
>
> - **`origin/HEAD` の利用条件**: リモートリポジトリをcloneしていれば通常は自動設定されていますが、機能しない場合は一度 `git remote set-head origin -a` を実行してローカルの参照を更新してください。
> - **`--keep-alive`**: ブラウザタブを閉じてもローカルサーバーを維持します。AIに修正させた後、リロードするだけで最新の差分を再確認できるため、AI開発ループにおいて必須のオプションです。
> - **`--clean`**: 起動時に過去のコメントや「確認済み（Viewed）」の状態を**すべてリセット**します。前回のレビューコメントを引き継いで翌日も作業したい場合は、手動で `npx difit . --keep-alive` のように `--clean` を外して起動してください。

## 2. 【AIプロンプト生成機能の極意】

`difit` を使って、人間がレビューした内容をハルシネーション（意図しないコード破壊）なくAIに修正させるための黄金サイクルです。

### 最速のレビュー＆AI修正フロー

1. **差分の確認**: ターミナルで `dif` （または `difm`）を実行し、自動で開くブラウザ画面を確認します。
2. **Suggestionブロックによる修正指示（※最重要）**: 修正したい行の「コメントボタン」を押します。ここで単に「変数名を変えて」と書くのではなく、コメント欄にある **Suggestion insertion button**（2026年2月実装）をクリックし、以下のようにGitHubライクな修正提案を直接記述します。
   Markdown
   ````
   ```suggestion
   const isActiveUser = user.status === 'active' && !user.isDeleted;
   ```
   フラグの判定ロジックが漏れているので、上記のように修正してください。
   ````
3. **プロンプトの一括コピー**: レビューが完了したら、キーボードの **`Shift+C`** を押します（または画面上の「Copy All Prompt」ボタンをクリック）。 これにより、「ファイルパス + 該当行番号 + あなたのコメント（Suggestion含む）」が構造化されたテキストとしてクリップボードに全コピーされます。
4. **AIエージェントへの指示出し**: CursorのChat/Composer、またはClaude Codeの入力プロンプトに、そのままペーストして送信します。AIはSuggestionブロックのコードを「その行にそのまま置換すればよい」と一意に解釈するため、周辺コードを壊すことなく1発で完璧な修正を行います。
5. **再確認**: AIが修正を終えたら、開いたままの `difit` のブラウザタブをリロード（再読み込み）し、修正が反映されているか確認します。問題なければコミットします。

## 3. 【直近半年のキラーアップデート活用法】

初心者から一気にパワーユーザーへ引き上げる、最新の強力な機能群です。

### ① Agent Skill の公式ネイティブ対応 (2026年3月実装)

Claude Code等のAgentic AIに、`difit`の存在をシステムレベルで教え込むことができます。

- **実務での活用**: プロジェクトルートで一度だけ以下のコマンドを実行します。
  Bash
  ```
  npx skills add yoshiko-pg/difit
  ```
  これを設定すると、AIが自律的に複数ファイルを書き換えた後、人間がターミナルを操作しなくても、AI自身が「差分を確認してください」とローカルサーバーを立ち上げてくれるようになり、シームレスな協業が可能になります。

### ② Comment List View とキーボードナビゲーション (PR #64)

マウスを使わずにレビューを完結させるための機能です。

- **実務での活用**:
  - ブラウザ上で **`j` / `k`** キーを押すと、変更行を上下に移動できます。
  - **`Shift+L`** を押すと「Comment list view」が開き、全ファイルに散らばったあなたのコメントだけが一覧表示されます。ファイル数が多いPRレビューの最終確認に最適です。

### ③ 自動折りたたみ機能（Auto-collapsed）

- **実務での活用**: `package-lock.json` などのロックファイルや、ツールが自動生成したファイル、削除されたファイル（Deleted files）は、UI上でデフォルトで折りたたまれた状態で表示されます。AIが大規模なリファクタリングを行って数百ファイルが変更された場合でも、人間が「本当に読むべきロジックの差分」だけが抽出されて表示されるため、認知負荷が劇的に下がります。
