## はじめに：なぜプロのエンジニアは「lazygit」を使うのか？

lazygitは、ターミナル上で動く視覚的なGitクライアントです。2026年現在、世界中のプロフェッショナル（GitHub Star数7万超）に愛用されており、モダンな開発現場におけるデファクトスタンダードの一つとなっています。

**AIを活用する現代において、lazygitが必須となる最大の理由は「細かいコミット（セーブ）の乱発」と「その後の爆速整理」にあります。**

AI（CopilotやCursorなど）にコードを書かせると、数十行のコードが一瞬で生成されます。しかし、それが一発で完璧に動くとは限りません。そのため、現代の開発では **「AIの提案でとりあえず動いた」「不要な行を少し削った」という極めて細かい粒度で、こまめにコミット（セーブポイントの作成）を行うのが最も安全な戦術** となります。

しかし、細かいコミットをそのままチームに共有すると「履歴が汚い」と怒られます。 lazygitを使えば、この **「散らかった細かいコミット群」を、マウスや矢印キーの直感的な操作だけで、一瞬にして「1つの綺麗なコミット」に圧縮・整理できます** 。Gitの黒魔術のような複雑なコマンド（`git rebase -i` など）を覚える必要はなく、脳のメモリをすべて「開発」に集中させることができます。

## 1. 【最強の config.yml】AI生成コードの差分を完璧に把握する

lazygitの動作を最適化する設定ファイルです。 （保存場所: Mac/Linuxは `~/.config/lazygit/config.yml`、Windowsは `%APPDATA%\lazygit\config.yml`）

以下の設定は、AIが生成したコードの「どこがどう変わったか」を見落とさないための構成です。そのままコピー＆ペーストして使用してください。 _(※エディタ上で `paging` や `subprocess` に警告の波線が出ることがありますが、ツールの仕様上のダミー警告ですので無視して構いません)_

YAML

```
# yaml-language-server: $schema=https://raw.githubusercontent.com/jesseduffield/lazygit/master/schema/config.json

gui:
  nerdFontsVersion: "3"                  # モダンなアイコンを表示し、画面の視認性を高める
  showFileIcons: true                    # ファイルの種類（TS, Pythonなど）を一目で判別
  border: rounded                        # 長時間見ても疲れにくい丸みを帯びたUI枠
  wrapLinesInStagingView: true           # 長いAI生成コードも画面の端で折り返して表示
  showBottomLine: true                   # 画面下部に常にショートカットキーのヒントを表示（初心者必須）

git:
  paging:
    colorArg: always
    # 【最重要】deltaを使用して、コードの変更を左右に並べて（side-by-side）美しく表示する
    pager: delta --dark --paging=never --line-numbers --side-by-side

  # AIによる文脈の飛躍を見抜くため、変更箇所の前後5行（デフォルトは3行）を表示する
  diffContextSize: 5

customCommands:
  # 実例: AIに変更内容を読ませて、コミットメッセージを自動生成させる
  - key: '<c-g>' # Ctrl + g キーで発動
    description: 'AIコミットメッセージ自動生成'
    context: 'files'
    subprocess: true
    # ステージングされた差分をAI（ここではaichat等のCLIツールを想定）に渡し、コミットを作成する
    command: >
      git diff --cached | aichat "この差分から、Conventional Commitsに準拠した簡潔なコミットメッセージを英語で1つだけ出力して" > /tmp/msg &&
      git commit -F /tmp/msg
```

### なぜこの設定が必要なのか？

AIは時に、既存のロジックの前後関係を無視したコードを出力します。高機能な差分表示ツールである **`delta`** を連携させ、画面を左右に分割（`--side-by-side`）して比較することで、「左（古いコード）と右（AIが書いた新しいコード）」の違いがひと目で分かり、意図しないバグの混入を視覚的に防ぐことができます。

> **事前準備（必須）:**
>
> - `delta` のインストールが必要です（Macなら `brew install git-delta`）。
> - カスタムコマンドの `<c-g>` を使うには、`aichat` 等のAI CLIツールがインストールされ、APIキーが設定されている必要があります。

## 2. 【パワーユーザーの必須キーバインド】AIの残骸を整理する神操作

lazygitは画面がいくつかのパネル（Files, Branches, Commitsなど）に分かれています。パネル間の移動は `1`〜`5` の数字キー、または `←` `→` キー、あるいはマウスクリックで行えます。

基本的な操作（ファイルを選んで `Space` でステージング、`c` でコミット）に慣れたら、以下の3つの「圧倒的な差がつく操作」をマスターしてください。

### ① AIの余計な変更を弾く「行単位のステージング」

AIが勝手に追加したデバッグ用の `print` 文などを除外し、綺麗なコードだけをコミットします。

1. `Files` パネルで変更されたファイルを選択し、**`Enter`** を押す（差分の詳細画面に入ります）。
2. 変更の塊（ハンク）ごとに移動できるので、本当に必要な箇所で **`Space`** を押して部分的にステージング（保存対象に追加）します。
3. さらに細かく「この1行だけ」を選びたい場合は **`v`** を押して選択モードに入り、矢印キーで選んで **`Space`** です。

### ② コミット履歴を一瞬でまとめる「Squash（スクワッシュ）」

AIと対話しながら「修正1」「修正2」「やっぱり戻す」と乱発した汚い履歴を、他人にレビューしてもらう前に「1つの美しいコミット」に圧縮します。

1. `Commits` パネルに移動します。
2. まとめたい古いコミットたちを **`v`** キーと **`↓`** キーで複数選択します。
3. **`s`** キーを押すと、選択したコミット群が一瞬で1つに融合（Squash）されます。

### ③ 失敗を恐れない最強のセーフティネット「Undo」

初心者がGitで最も怖いのは「操作を間違えてコードが消えること」です。

- 間違えてコミットした、間違えて変更を破棄したなど、あらゆる場面で **`z`** キーを押してください。
- 「直前のGit操作」を安全に取り消す（Undo）ことができます。

## 3. 【AIとGitの統合】思考を止めないコミット生成

前述の `config.yml` に設定したカスタムコマンドの活用法です。

AIによってコードを書くスピードが劇的に上がった反面、「この大量の変更をどう要約してコミットメッセージに書くか」で手が止まってしまうエンジニアが増えています。 ファイルをステージングした後に **`Ctrl + g`** を押すだけで、AIが差分を読み取り、適切なメッセージをつけて自動でコミットしてくれます。これにより、タイピングの手間を省き、開発のフロー状態を維持したまま次の実装へ移ることができます。
