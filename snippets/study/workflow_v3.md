# AI駆動型学習ワークフロー v3（教材非依存・運用版）

> **目的**: NotebookLM を「根拠の母艦」にしつつ、Gemini / Claude を使って知識を「実務の武器」に変える
>
> **設計思想**: 用途（自分用/共有用）でレーン分離し、ノイズ混入と“それっぽい誤り”を避ける

---

## 概要

本ワークフローは、以下の5ステップで構成します。

1. **ステップ0（準備）**: 教材の電子化と、NotebookLM/Geminiの初期設定
2. **ステップI（インプット・戦略）**: Video/Audioで理解の初速 → 戦略地図（学ぶ順/捨てる順）を作る
3. **ステップII（集中学習）**: 読書＋チャット＋難所突破＋図解で理解を固める
4. **ステップIII（武器化・定着）**: 実務接続の壁打ち → 自己テスト
5. **ステップIV（共有・資産化）**: 共有するならSpec→編集可能成果物にする

---

## 大原則：生成物の“用途”でレーンを分ける

- **理解の初速（自分用）**: NotebookLM Studio（Video/Audio/Slide Deck/Infographic）でOK
  - 崩れ・誤り・編集性の低さは許容（あくまで学習補助）
- **共有・資産化（編集前提）**: NotebookLMは「表（Data Table）/設計図（Spec）」まで
  - 最終成果物は **Gemini Canvas / Claude Artifacts** で作る
  - 共有用の再現手順は本書のステップIVを参照（スライド/インフォグラフィック）

---

## ステップ0: 準備（初回だけ）

### 0-1. 教材の電子化（PDF/OCR/分割）

- **紙の本**: スクショ→PDF化→OCR（検索できる状態にする）
- **PDFが重い/長い**: 章/トピック境界で分割（後でソースON/OFFしやすくする）
  - 例: 章ごと / テーマごと（セキュリティ/並行処理/料金…）

### 0-2. NotebookLM ノートの初期化（manifest運用）

NotebookLMのソースに、**常時ONの manifest**（目的・優先テーマ・出力ルール）を置きます。

- **目的**: 「このノートで最終的に何を作るか」を固定し、脱線と平均化を減らす
- **推奨ルール**:
  - 「今チェックされているソースのみ」を根拠にする（推測禁止、不明は不明）
  - 出力は「章/ソース」単位で分ける
  - 共有用の成果物は **Data Table / Spec（設計図）** までにする

### 0-3. Geminiの“モード制”を入れる（重要）

Geminiは強い分、**教材の記述**と**一般知識/最新情報**が混ざると監査不能になります。
カスタム指示を入れて、回答を3モードで分離します。

- **設定ファイル**: [GemNLMアナリスト・メンター.md](prompt/GemNLMアナリスト・メンター.md)
  - 忠実/統合/最新（Web）の宣言＋段落分離＋ノイズ対策（参照候補を先に絞る）

### 0-4. 最新仕様を調べる（必要な時だけ / 任意）

教材（NLM）と最新（Web）を混ぜると事故るので、最新モードの調査は「別レーン」で実施します。

- **ツール**: Gemini（Deep Research）
- **プロンプト**: [Gem用\_DeepResearch調査.md](prompt/Gem用_DeepResearch調査.md)
- **出口**: 結果は Data Table に「Web由来（取得日つき）」として追記して残す
- **注意**: Deep Researchは時間がかかることがあるので、実行する場合は「やる/やらない」を先に決める。

## ステップI: インプット・戦略設計フェーズ

学習開始時に、**全体像と戦略**を掴みます。

### I-1. 視覚・聴覚インプット（NotebookLM Studio / 自分用）

#### 動画解説（基本：メイン）

- **ツール**: NotebookLM Video Overview
- **ビジュアル設定**: [NLM動画解説カスタムスタイル.md](prompt/NLM動画解説カスタムスタイル.md)
- **プロンプト**: [NLM動画解説プロンプト3.md](prompt/NLM動画解説プロンプト3.md)
- **目的**: マクロ→ミクロで全体像を掴む（理解の初速）

#### 音声解説（ながらオプション：サブ）

- **ツール**: NotebookLM Audio Overview
- **プロンプト**: [NLM音声解説プロンプト.md](prompt/NLM音声解説プロンプト.md)
- **目的**: 移動中など“画面を見ない前提”でインプット

#### スライド解説（時短オプション）

- **ツール**: NotebookLM Slide Deck
- **プロンプト**: [NLMスライド資料プロンプト.yml](prompt/NLMスライド資料プロンプト.yml)
- **目的**: 動画を見る時間がない時に、自分のペースで高速に要点を把握
- **注意**: 共有・再利用前提なら **ステップIV（Spec→編集可能）** を推奨

### I-2. 戦略的全体分析（Gemini Canvas）

- **ツール**: Gemini Canvas
- **プロンプト**: [GemNLM分析（Canvas）.md](prompt/GemNLM分析（Canvas）.md)
- **目的**: 章/トピックごとに「手札/嗜み/スキップ」を作り、読む順を決める

## ステップII: 集中学習フェーズ

教材を読み進めながら、AIで理解を深めます。

### II-1. 通常の読書とチャット

- **ツール**: Gemini（カスタム指示設定済み）
- **ポイント**: 質問のたびに「忠実/統合/最新」を宣言して混ぜない

### II-2. 難所突破（深掘りモード）

- **プロンプト**: [難しい概念を解説.md](prompt/難しい概念を解説.md)
- **目的**: 「歴史的背景/トレードオフ/物理イメージ」で腹落ちさせる

### II-3. 視覚化（構造がごちゃごちゃした時 / オプション）

#### インタラクティブHTML図解

- **プロンプト**: [概念の図解化.md](prompt/概念の図解化.md)
- **目的**: 関係性や構造を可視化（ブラウザでインタラクティブに操作可能）

#### インフォグラフィック（一枚絵）

- **ツール**: NotebookLM Infographic
- **プロンプト**: [NLMインフォグラフィック.md](prompt/NLMインフォグラフィック.md)
- **設定**: レイアウト「横向き」、詳細レベル「詳細（ベータ版）」
- **目的**: GoodNotesに直接貼り付けて保存する用
- **注意**: 共有・再利用前提なら「設計図（InfographicSpec）→ HTML（編集可能）」を推奨（ステップIV参照）

### II-4. “運用する表”に落とす（Data Table）

**ここが“武器化”の本体**です。運用は Data Table に統一します。

- **やること**:
  - 今日のテーマに対応する Data Table を開く
  - **ソースチェックを今日の範囲に絞る（重要）**
  - 生成し直す or 差分更新
  - 可能なら Sheets にエクスポートして、運用ログ列（自分の例/試した日/結果）を追加
- **表の型（例）**:
  - 索引: Term / Type / Definition / WhenToUse / Pitfall / Source / Section / Confidence
  - 判断表: テーマ / 結論 / 判断基準 / 具体例 / リスク/落とし穴 / 推奨アクション / 根拠
  - 診断表: 症状 / 原因候補 / 対処手順 / 再発防止 / 根拠
  - 安全運用: テーマ / 結論（やる/やらない/条件付き）/ ガードレール / 事故パターン / 事前チェック / 根拠

---

## ステップIII: 武器化・定着フェーズ

### III-1. ガイド付き深掘り学習

- **ツール**: Gemini（ガイド付き学習）
- **プロンプト**: [GemNLM質問（ガイド付き学習）.md](prompt/GemNLM質問（ガイド付き学習）.md)
- **目的**: 実務での使いどころ（判断根拠）に変換する

### III-2. 武器化完了チェック（セッション終了時）

NotebookLMを参照せずに、次を短く答える。

1. **実務接続**: 今日学んだ〇〇を、明日の仕事で使うとしたらどの場面で何の判断根拠になるか？
2. **Before/After**: 知らなかった頃はその場面でどう判断していたか？
3. **30秒説明**: 同僚に30秒で説明するならどう言うか？

## ステップIV: 共有・資産化フェーズ（必要な時だけ）

### IV-1. スライド資料（編集可能）

**再現手順（推奨）**:

- **(1) SlideSpecを作る（設計図）**: [NLM用\_スライド設計図出力.md](prompt/NLM用_スライド設計図出力.md)
- **(2) スライドを描画する（編集可能な成果物）**: [Gem用\_スライド描画実行.md](prompt/Gem用_スライド描画実行.md)

**使い分け（目安）**:

- **Googleスライド運用**: Gemini Canvas（見栄えが安定しやすい）
- **.pptx化したい**: Claude Artifacts も候補

### IV-2. インフォグラフィック（編集可能HTML）

**再現手順（推奨）**:

- **(1) InfographicSpecを作る（設計図）**: [NLM用\_インフォグラフィック設計図出力.md](prompt/NLM用_インフォグラフィック設計図出力.md)
- **(2) HTMLを生成する（編集可能な成果物）**: [Gem用\_インフォグラフィック描画実行.md](prompt/Gem用_インフォグラフィック描画実行.md)

**使い分け（目安）**:

- HTML生成は Claude Artifacts の方が安定する傾向（経験則）
- Gemini Canvasでも可能だが、崩れが出たらArtifacts側に寄せる

### IV-3. PDF保存（知識資産化）

- **プロンプト**: [テキストをHTML資料化.md](prompt/テキストをHTML資料化.md)
- **目的**: 対話ログやメモをHTMLに整形→PDF保存（GoodNotes等へ）

## クイックリファレンス

### 日常の学習フロー（最小構成）

```
1. NotebookLM（Video or Audio）で理解の初速（5-10分）
2. Gemini Canvasで戦略地図（学ぶ順/捨てる順）（10分）
3. 読書 + Geminiチャット（忠実/統合/最新を分ける）
（必要な時だけ）最新モードは Deep Research で別レーン調査 → Data Tableへ反映
4. Data Table更新（根拠つきで武器にする）
5. 3つの問いで自己テスト（5分）
```

| 状況                           | 使うもの                                                                                                                                                                          |
| :----------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| 関係性がごちゃごちゃ           | [概念の図解化.md](prompt/概念の図解化.md)                                                                                                                                         |
| 一枚絵で俯瞰したい（自分用）   | [NLMインフォグラフィック.md](prompt/NLMインフォグラフィック.md)                                                                                                                   |
| 共有用スライドを作りたい       | [NLM用\_スライド設計図出力.md](prompt/NLM用_スライド設計図出力.md)→[Gem用\_スライド描画実行.md](prompt/Gem用_スライド描画実行.md)                                                 |
| 共有用インフォグラフィック     | [NLM用\_インフォグラフィック設計図出力.md](prompt/NLM用_インフォグラフィック設計図出力.md)→[Gem用\_インフォグラフィック描画実行.md](prompt/Gem用_インフォグラフィック描画実行.md) |
