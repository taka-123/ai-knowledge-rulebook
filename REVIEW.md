# REVIEW.md

このファイルは、PR・差分レビュー用の方針テンプレートです。
実装作業の通常ルールは `AGENTS.md` に従い、レビュー時のみ本ファイルを参照します。
コピー後は、使用するリポジトリの実態に合わせて空欄を埋め、不要な項目や例は削除してください。

## 基本方針

- レビューコメントは日本語で書く。
- 識別子、設定キー、コード、ログ、引用は原文を維持する。
- 差分によって新たに入った問題、または差分で悪化した問題を優先して指摘する。
- 実害、根拠、最小修正案を示せる場合だけ指摘する。
- 同一原因の指摘は集約する。
- 秘密値、トークン、個人情報、内部情報は再掲せず伏せる。
- P0 / P1 相当の指摘がない場合は `指摘なし` と返す。

## 重大度

Codex GitHub Review では、原則として P0 / P1 相当の問題だけを指摘する。
P2 / P3 は、ローカルレビューや人間向けレビューで必要な場合だけ使用する。

- `P0`:
- `P1`:
- `P2`:
- `P3`:

記入例（コピー後は削除または置換）:

- `P0`: 機密漏えい、重大なデータ破壊、認証・認可の重大な欠陥、本番停止級の問題
- `P1`: ユーザー影響の大きい誤動作、重大な仕様不整合、回避困難な障害、重要なテスト欠落
- `P2`: 保守性低下、将来の不具合につながる設計・実装上の問題
- `P3`: 体裁、一貫性、軽微な表現改善

## コメント形式

指摘には、可能な限り次を含める。

- 問題点
- 影響
- 根拠
- 最小修正案

形式:

```text
[P1] タイトル

問題:
影響:
根拠:
最小修正案:
```

- 未確認事項は `確認事項:` として分ける。
- 差分外の提案は `follow-up:` として分ける。
- 修正必須ではない好みの指摘は、その旨を明記する。
- 同じ修正で解消できる指摘は、複数コメントに分散させず集約する。
