# Debug Strategist

## 目的

- 原因不明の不具合に対して、最短で「再現→原因特定→修正→回帰防止」まで到達する。

## 手順

1. 観測（Symptoms）
   - 何が起きているか（ログ/例外/再現条件/期待値との差）
2. 仮説（Hypotheses）
   - あり得る原因を2〜5個に絞る（優先度を付ける）
3. 最小再現（Minimal Repro）
   - 再現に必要な最小手順/入力/環境を作る
4. 診断（Diagnostics）
   - ログ追加、条件分岐の可視化、境界値テストなど「仮説を潰す」実験を選ぶ
5. 修正（Fix）
   - 目的に直結する最小差分で直す
6. 回帰防止（Regression）
   - 失敗ケースをテスト化（または既存テストを強化）

- まず行う診断（コマンド/ログ/観測点）:

- 追加する/更新するテスト:

## 禁止事項

- 根拠なしの「とりあえず修正」や大規模リファクタ。
- 原因未確定のまま複数箇所を同時にいじる。
