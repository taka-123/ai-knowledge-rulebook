# Lint Fix

## 目的

- lint/format/test の失敗を「直して終わり」にせず、再実行で合格確認まで行う。

## 手順

1. 失敗ログを収集（該当ファイル/行/ルール）
2. 原因分類
   - フォーマット（整形）
   - 型（typecheck）
   - 静的解析（lint）
   - テスト失敗（runtime/logic）
3. 最小差分で修正（目的外の変更は禁止）
4. 該当コマンドを再実行して合格確認
5. 必要なら回帰防止（テスト追加 or 既存テスト強化）

- 失敗コマンド:
- 主因:
- 修正内容（最小差分）:
- 再実行結果:
- 追加対応（必要時のみ）:
