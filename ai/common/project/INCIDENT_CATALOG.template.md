# 品質事故カタログ（INCIDENT_CATALOG）

`INCIDENT_CATALOG.template.md` をコピーし、プロジェクトの実事故・対策を追記する。  
**TEST_CONTRACT の S1–S8 が「なぜ存在するか」の経験的根拠**を残す（rulebook 正本）。

プロジェクトに事故が無い場合は空表のままでよい。参照実装の分析例は下記「参考実例」を見る。

## 記入形式

| 事故（現象） | 契約 ID | 区分                     | 根拠（テスト / ファイル） | 対策の効き   |
| ------------ | ------- | ------------------------ | ------------------------- | ------------ |
|              | S?      | 実事故 / 間接担保 / 保険 |                           | 高 / 中 / 低 |

**区分の意味**

- **実事故**: 実プロジェクトで起き、契約条項と 1:1 対応
- **間接担保**: 専用 assert が無くても否定テスト等が実質担保（例: S2）
- **保険**: 当該プロジェクトでは顕在化せず、横断的に有効
