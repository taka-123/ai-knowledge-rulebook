# TRACEABILITY — 要件 ↔ 実装 ↔ 検証（監査成果物）

**常時維持する生きた表ではない。**  
`spec-auditor` がマイルストーン / 機能完了前の監査で**生成・出力**する成果物の雛形。

- spec-auditor は**監査レポート本文**として返す。`docs/audits/` への保存は呼び出し側
- 保存先の例: `docs/audits/TRACEABILITY-M[N].md`
- 否定検証列が空、または共通レイヤー参照が無効 → **監査 FAIL**

## 記入形式

| 要件引用（原文）      | 実装根拠（ファイル / API / data 属性） | 肯定検証 | 否定検証（または共通レイヤー参照）  | 状態    |
| --------------------- | -------------------------------------- | -------- | ----------------------------------- | ------- |
| [SPEC / チケットから] | [path:line 等]                         | [名前]   | [名前 または `→ interact E2E §xxx`] | ⬜ / ✅ |
