# マルチサービス統合プロジェクト ガイドライン

> Motto: "サービス境界を越えた整合性と、子リポジトリへの即座解決"

## 1. 最優先トリガー

- ユーザーが子サービス名を言及したら、即座に `{サービス名}/AGENTS.md` を読んでコンテキストを同期する。
- 直接パス指定を優先し、パス不明時のみ最小限の探索を行う。

## 2. 共有ソース（必要時のみ）

<!-- テンプレート注記: `README.md` / `directorystructure.md` / `technologystack.md` の3項目のみ使用。不存在の行を削除し、3項目すべて不存在なら本セクションを削除 -->

- 実態との乖離が疑われる場合、ドキュメントを鵜呑みにせず実ディレクトリ・設定と照合する。
- **全体概要**: `README.md`
- **技術スタック/バージョン**: `technologystack.md`
- **ディレクトリ構造/命名規則**: `directorystructure.md`

## 3. サービス横断ルール

- 作業対象ディレクトリに `AGENTS.md` がある場合は、その個別ルールを本ファイルより優先する。

## 4. 運用補足（任意）

- 共通コマンドは必要時のみ追加する（全サービス共通で毎ターン有益な操作に限定）。
