# markdown-line-length

Markdown の行長を保ち、ソースとレンダリング両方で読みやすくする。

## 行長の目安

- 80〜120 文字を目安にする。厳密上限ではない。
- URL・テーブル・コードブロック・見出しは超過可。

## 平文（段落テキスト）

句点（。）や意味の切れ目で改行してよい。Markdown の単一改行はレンダリング時にスペース扱いとなり、表示に影響しない。

## 箇条書き（最重要）

**リスト項目の途中で物理改行を入れてはならない。** Markdown ではリスト項目の途中改行はレンダリング時に 1 行へ結合される。見た目は変わらないのに、ソースだけが不自然に分断される。

長い項目への対処（優先順）:

1. **ネストで分割** — 子項目に分けて構造化する
2. **文を短く書き直す** — 冗長な表現を削る
3. **そのまま残す** — 120 文字程度なら無理に短縮しない

NG（途中で改行している例 — レンダリングでは 1 行に結合される）:

```markdown
- 既存のAIレビューエージェント
  （review-core-diff-gatekeeper等）を
  GitHub Actions CI に組み込む。
```

OK（ネストで構造化）:

```markdown
- AIレビュー自動化のCI統合
  - review-core-diff-gatekeeper等をGitHub Actionsに組み込む
  - PR作成時に一次レビューを自動実行
```

## 見出し

短ければ 1 行のまま。極端に長い場合のみ短縮する。
