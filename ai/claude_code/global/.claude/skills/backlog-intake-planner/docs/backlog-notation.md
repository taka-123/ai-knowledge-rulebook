# Backlog 記法（requirements.md 出力用）

`requirements.md` は Backlog にそのまま貼って崩れない形で出力する。以下の規則に従う。
（`implementation-plan.md` は対象外。こちらは標準 GFM でよい＝Backlog に貼らない前提。）

## 規則

1. 箇条書きは `* ` を使う（`- ` は使わない）。
2. ネストは**半角スペース4つ**で統一する（タブ・2スペース禁止）。例: `    * 子項目`。
3. チェックリストは `* [ ] ` 形式。
4. 番号付きリストは `1. ` 形式。
5. 見出しは ATX 形式（`#`）。
6. 引用は各行 `> ` で始める。空行は `>` 行で表す。
7. コードはフェンス付きコードブロック（3連バッククォート）。
8. 罫線は `---` に統一する。
9. 強調はアスタリスク（`*italic*` / `**bold**`）。
10. リンクは `[text](url)`、画像は `![alt](url)`（title 属性は付けない）。
11. 表は `|` を使う GFM 形式。
12. 定義リスト・`<details>` は通常テキストへ展開する。
13. Backlog 固有記法（`BLG-95`, `[[WikiPage]]`, `#rev(...)`）は保持する。

## 注意

- 一般的な Markdown 慣習より、上記規則を優先する。
- 入力に `-` と `*` が混在していても、出力は `*` に統一する。
