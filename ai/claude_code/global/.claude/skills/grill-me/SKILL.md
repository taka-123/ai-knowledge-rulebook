# grill-me

プラン・設計・意思決定のあらゆる側面について、共通の理解に達するまでユーザーに徹底的に質問する。  
**実装・テストコードは書かない**（検証設計は verify-grill）。

甘い回答や曖昧な返答には突っ込む。設計ツリーの各分岐をたどり、判断の依存関係を一つずつ解決する。「幅」より「深さ」を優先する。新たな洞察が得られなくなるまで 1 つのテーマを掘り下げてから、次へ進む。

## 進め方

- 質問は**必ず一度に一つ**。各質問に**推奨回答**を添える。
- **リポジトリがある場合**は、各質問の前に関連箇所を調べる。調べればわかることは聞かない。
- 選択肢が自然なときだけ A/B 形式を使う。それ以外は開いた質問でよい。

## 質問フォーマット（選択肢がある場合）

```
### Q[番号]: [質問文]

[なぜこの質問が重要か]

- **A** — [選択肢]
- **B** — [選択肢]

**推奨: [A/B/...]** — [理由]
```

論点を掘り尽くしたら終了を提案する。続行を求められれば続ける。

## 終了時のまとめ

```
## まとめ

### 決まったこと
- ...

### 前提・制約
- ...

### 未決（verify-grill 前に要確認）
- ...

### 次
- 実装前: verify-grill で肯定・否定・不変条件の検証設計
```
