# instruction-pruning

AI 向け指示文書から、書かなくても挙動が変わらない指示を削る。
残してよい指示は 2 種類だけ。

1. **固有規約** — モデルが推測できないもの（環境固有の記法・パス・フォーマット・SOP・好み）
2. **反デフォルト規律** — 指示がないと有能なモデルが流されやすい方向への逆張り（実際に起きた / 起きうる失敗に根拠があるもの）

## Procedure

1. 各指示に判定テストを適用する: **この行を消したら、有能なモデルの挙動が変わるか**。変わらないなら削除候補。
2. 迷ったら根拠で決める: その失敗の具体例を挙げられるなら残す。挙げられないなら削る。
3. 重複は判定ロジック側の 1 箇所に寄せる（description と本文、判定表と禁止リストの二重化を解消）。
4. 監査依頼では削除候補を根拠付きで提示する。編集依頼では削除まで行う。

## Prune patterns

| パターン                       | 例                                                                     |
| ------------------------------ | ---------------------------------------------------------------------- |
| 既定挙動の言い直し             | 「事実を捏造しない」「丁寧に確認する」                                 |
| 存在しない懸念への予防         | 言及されなければ誰もやらない操作の禁止                                 |
| 仕組みが保証済みのことの念押し | disallowedTools で塞がれた操作の禁止、判定表が除外済みのケースの再禁止 |
| 自動読込ファイルへの参照誘導   | 毎ターン読み込まれる恒久指示ファイルへの「参照しろ」                   |
| 設計意図の解説                 | ルールの直後に続く「本スキルは〜を定めるものであり…」                  |

## Exceptions

- セキュリティ・破壊的操作のガードレールは、自明でも残してよい（失敗コストが非対称）。
