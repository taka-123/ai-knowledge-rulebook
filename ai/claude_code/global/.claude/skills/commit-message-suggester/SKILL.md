# commit-message-suggester

未コミット差分**全体**を対象に、`<prefix>: <日本語1行>` のコミットメッセージと、差分の扱い（1 コミットにするか分割するか）を提案する。

## Procedure

1. 未コミット差分（staged 含む）の**全体**を確認する。直前の作業分だけを対象にしない。
2. 差分を意図単位に分ける。1 意図なら 1 コミット。複数意図なら、各コミット単位に対象ファイルと順序を添えた分割案を提示する。
3. 各コミット単位にメッセージを 1 つ提案する。prefix で迷う場合は候補 2 つを根拠 1 文付きで併記する。
4. コミットを実行するかは下記「実行モード」に従う。

## フォーマット

| prefix     | 用途                                               |
| ---------- | -------------------------------------------------- |
| `feat`     | 新機能・新規ファイル/スキル/エージェント追加       |
| `fix`      | バグ修正・不具合対応                               |
| `refactor` | 挙動を変えない構造整理・書き直し                   |
| `chore`    | 設定整理・依存更新・ドキュメント更新などの周辺整備 |

ルール:

- `<prefix>` の直後は **半角コロン + 半角スペース 1 個**。
- 日本語 1 行、改行・句点（`。`）禁止。

## 実行モード

- **対話作業中（既定）**: メッセージと差分の扱いの提案までで止める。コミットは明示依頼があってから実行する。
- **自律作業中**: ユーザーがコミット（またはプッシュ）まで含めて任せている場合は、提案で停止せず、本フォーマットのメッセージでそのままコミットする。

## 提案タイミング（自発）

次の区切りでコミットを自然に提案する。同じ区切りで繰り返さず、作業の流れを止めない。

- 検証（テスト・lint 等）が通り、1 つの意図の作業が完了したとき
- 意図の異なる作業へ移る前（差分に複数意図が混ざり始める前）

## Examples

- `feat: commit-message-suggester スキルを追加`
- `fix: 認証ハンドラで null セッションを参照しないよう修正`
- `refactor: 種別判定を表駆動に整理`
- `chore: グローバルスキルの記述を精選`

複数意図混在時（未コミット差分全体を見た分割提案）:

```text
未コミット差分に複数意図が混ざっています。以下の分割を推奨します。
1. fix: 既存ハンドラの null 参照を修正 — src/auth.ts
2. feat: 通知メール送信機能を追加 — src/notify.ts, src/templates/
```
