# conversation-graphic-recorder

現在利用できる会話・資料・差分から、最終的に重要な内容だけを抽出し、
日本語のグラフィックレコーディング画像として1枚に図解・可視化する。

目的はログの転記ではない。
元の情報を読み返さなくても、結論、判断理由、変更点、関係性、流れ、次の行動を把握できる状態にする。

## Procedure

1. **実行可否を確認する**
   - 現在の環境に実際の画像生成機能がある場合だけ続行する。
   - 画像生成機能がない場合は、画像を生成できないことを短く伝えて終了する。
   - ユーザーが明示的に求めた場合だけ、画像生成機能を持つ別のエージェント環境へ渡すための生成指示をテキストで作る。

2. **対象を確定する**
   - 現在の会話
   - ユーザーが指定した資料
   - Git差分（指定がある場合）
   - ユーザーが明示した範囲を最優先する。

3. **向きを決める**
   - 指定なし: 横長 `16:9`
   - 「横長」「landscape」「horizontal」: 横長 `16:9`
   - 「縦長」「portrait」「vertical」: 縦長 `4:5`
   - 向きだけを変更する場合、結論、事実、判断、基本的な雰囲気は維持し、配置と文章量だけを再構成する。

4. **情報を選別する**
   次の順で残す。
   1. 内容を正確に表すタイトル
   2. 最重要メッセージ
   3. 最終結論
   4. 重要ポイント3〜6個
   5. 判断理由、変更点、比較、関係性、手順または時系列
   6. 次に行うこと
   7. 再評価条件または未確定事項
   8. 必須の固有名詞、日付、数値、製品名

   次は除外する。
   - 訂正前・撤回済みの情報
   - 途中の試行錯誤
   - 謝罪、言い直し、会話上の弁明
   - 結論に影響しない重複

   明確な結論がない場合は、結論を作らず「暫定結論」と「未確定事項」を分ける。

5. **内部レイアウトを1つ選ぶ**
   ユーザーに分類名を覚えさせず、内容の中心的な問いから自動選択する。

   ### 判断・比較

   使用対象:
   - 複数案の比較
   - 採用・不採用
   - 方針決定
   - ツール選定
   - Before / After
   - Git差分の「何が変わったか」

   基本構成:
   - 比較対象または変更前
   - 判断基準または主要な変更点
   - 最終結論または変更後
   - 影響、条件、次の行動

   ### 構造・関係

   使用対象:
   - 役割分担
   - 依存関係
   - システム構成
   - 原因と影響
   - 制約と打ち手

   基本構成:
   - 中心テーマ
   - 主要要素
   - 関係を示す矢印や線
   - 必要に応じて原因、影響、制約、打ち手

   ### 流れ・手順

   使用対象:
   - 作業手順
   - 導入・移行
   - 時系列
   - ロードマップ
   - 課題から解決までの経緯

   基本構成:
   - 現在地または開始点
   - 3〜6個の主要ステップ
   - ゴールまたは最終状態
   - 注意点と次の一歩

   ### 全体要約

   使用対象:
   - 学習内容
   - 複数の論点を含む会話
   - 比較・構造・流れのいずれも中心ではない内容

   基本構成:
   - 核心
   - 3〜6個の重要ポイント
   - 必要な箇所だけ矢印、番号、比較
   - まとめまたは次の行動

   選択優先順位:
   1. ユーザーが自然文で指定した見せ方
   2. 「どれを選ぶか」「何が変わったか」なら判断・比較
   3. 「どう関係するか」「どう構成されるか」なら構造・関係
   4. 「どう進めるか」「どう変化したか」なら流れ・手順
   5. それ以外は全体要約

6. **画像を生成する**
   - 1枚絵
   - 日本語
   - 手描き風グラフィックレコーディング
   - 白または淡い紙色の背景
   - 太めで読みやすい線
   - 色は3〜5色程度
   - 写真風・3D風ではなく、図解と簡単なアイコンを中心にする
   - 情報ブロックは3〜6個
   - 装飾より情報理解と視線誘導を優先する
   - 最後に「まとめ」「次のアクション」「次の一歩」のいずれかを置く

## Orientation rules

### 横長

- 左から右への流れを基本とする
- 比較、関係図、手順、Before / Afterに適用しやすくする
- 情報ブロックは最大6個程度

### 縦長

- 上から下への流れを基本とする
- スマートフォンで読める文字サイズを優先する
- 情報ブロックは3〜5個を目安に絞る
- 横長版を単純に縮小せず、文章量と配置を再設計する

## Text rules

- 長文を入れず、短い見出しと短いフレーズへ圧縮する
- 文字は大きくする
- 固有名詞、数字、日付、製品名、専門用語は可能な限り正確に保持する
- 一つのテキスト要素を短くし、誤字のリスクを下げる
- 英語が正式名称の場合は英語を保持する
- 入力にないキャッチコピーを作らない

## Prohibited

- 入力にない事実、評価、因果関係、結論を追加する
- 訂正前や撤回済みの内容を最終結論として扱う
- 長文や小さい文字を大量に入れる
- 装飾を増やして読みにくくする
- 意味のないチャートを作る
- 同じ内容を複数箇所へ重複させる
- 内部レイアウト名を画像内に表示する
- 画像生成機能がない環境で、生成できたように装う

## Output

- 原則1枚だけ生成する。
- 複数案は、ユーザーが明示的に求めた場合だけ生成する。
- 生成前に長い説明や構成案を返さない。
- 画像生成後に長い解説を追加しない。
