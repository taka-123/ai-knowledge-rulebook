# Conversation Graphic Recorder — Style Guide

このファイルは、会話、調査、学習、意思決定、Git差分を日本語グラレコへ変換するときの
スタイル参照資料である。固定テンプレートではなく、内容に合う型を選ぶために使う。

## 既定

- 既定比率: 横長16:9
- 既定スタイル: 内容から自動選択
- 判断できない場合: `general`
- 縦長指定: `portrait` / `vertical` → 4:5
- 横長指定: `landscape` / `horizontal` → 16:9

## スタイル選択

| 内容                             | スタイル   | 推奨比率       |
| -------------------------------- | ---------- | -------------- |
| 比較、採否、方針、意思決定       | `business` | 16:9           |
| Git差分の変更要約、変更前後      | `business` | 16:9           |
| 講義、書籍、学習、概念           | `learning` | 16:9 または4:5 |
| 手順、計画、成長、移行           | `roadmap`  | 16:9           |
| 因果関係、構造、依存関係         | `system`   | 16:9           |
| アーキテクチャ・データフロー変更 | `system`   | 16:9           |
| 会議、イベント、議論             | `live`     | 16:9           |
| 課題、気づき、変化、結果         | `story`    | 16:9           |
| SNS、スマートフォン、保存用      | `magazine` | 4:5            |
| その他                           | `general`  | 16:9           |

## 安定させる原則

- タイトルは1つ。
- 最重要メッセージは1つ。
- 情報ブロックは3〜6個。
- 文字は短いフレーズ中心。
- 必ず残す固有名詞、数字、日付を優先する。
- 入力にない事実や因果関係を追加しない。
- 日本語の長文を画像内へ入れない。
- 3〜5色、白または淡い背景、太めの線。
- 装飾より視線誘導と理解を優先する。
- 最後に、まとめ、次のアクション、次の一歩のいずれかを置く。

## general

- 横長を標準とする。
- 中央に全体テーマ。
- 周囲に3〜6個の要点。
- 関係は矢印、線、番号で示す。
- 右下にまとめまたは次のアクション。

## business

- 左: 現状、変更前、課題。
- 中央: 判断の核心、主要変更、方針。
- 右: 変更後、解決策、採用内容。
- 下: 影響、条件、次のアクション。
- 落ち着いた3〜4色。
- 信頼感のある資料風。

Git差分では次の構成を優先する。

- 上: 変更全体の要約。
- 左: 変更前または対象領域。
- 中央: 実際の主要変更3〜5個。
- 右: 新しい振る舞いまたは影響。
- 下: staged / unstaged / untracked と次の確認。

## learning

- 中央: 今回の核心。
- 周囲: 重要概念4〜6個。
- 必要に応じて、比較、プロセス、因果関係を図示。
- 下: 覚えておくこと、次に試すこと。
- 手描きノート風。

## roadmap

- 左下: 現在地。
- 右上: ゴール。
- 途中: 3〜6個のチェックポイント。
- 課題: 山、谷、注意マーク。
- 支援策: 橋、道具。
- 最後: 次の一歩。
- ファンタジーに寄せすぎず、業務にも使える表現。

差分では、旧状態から新状態への段階的移行やマイグレーションの説明に使う。

## system

- 中央: 中心課題、中心機能、または最重要概念。
- 周囲: 原因、影響、関係者、制約、打ち手、コンポーネント。
- 関係を矢印で結ぶ。
- ループがある場合は循環矢印で示す。
- 下: レバレッジポイント、主要変更、次の確認。
- アイコンより構造の読みやすさを優先する。

差分では、コンポーネント間の接続、データフロー、依存関係の変更に使う。

## live

- 左: 背景、課題。
- 中央: 議論の核心。
- 右: アイデア、解決策、決定。
- 下: 気づき、まとめ、次のアクション。
- ホワイトボード上にその場で描いたようなライブ感。
- 人物間の意見は吹き出しで示す。
- 発言者名は必要な場合だけ使い、不要なら役割名にする。

## story

- 4〜6コマ。
- 左上から右下へ読む。
- 基本構成: 現状 → 困りごと → 気づき → 解決策 → 変化 → まとめ。
- 登場人物は最小限。
- 入力にない人物設定を追加しない。

## magazine

- 縦長4:5を標準とする。
- 上: 大きなタイトルと短いリード。
- 中央: メイン図解。
- 周囲: 3〜5個の要点カード。
- 下: 「つまり何が大事？」またはまとめ。
- 洗練された誌面風だが、文字は少なく保つ。

## 差分可視化の追加ルール

- コード行を画像へ大量に載せない。
- ファイル名は主要なものだけ。
- ファイル単位より、機能、層、責務でまとめる。
- 「変更前」と「変更後」を混同しない。
- 追加、変更、削除を色や記号で区別してよい。
- 品質評価を求められていない場合、良い・悪いの判定を付けない。
- 差分から断定できない影響は「推定」と明記する。
- staged、unstaged、untrackedの区別が重要なら下部へ状態として示す。

## 任意指定の例

- `create`: 自動選択、16:9、auto backend
- `portrait`: 自動選択、4:5、auto backend
- `landscape business`: ビジネス資料型、16:9
- `portrait magazine`: 雑誌インフォグラフィック型、4:5
- `landscape system`: システム思考型、16:9
- `landscape roadmap`: ロードマップ型、16:9
- `diff landscape business`: 現在の差分を変更前後で整理
- `diff svg`: ネイティブ画像生成を使わずSVGで生成
