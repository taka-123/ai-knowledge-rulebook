# Native Image Generation Prompt Template

ネイティブ画像生成ツールへ渡すプロンプトは、次の構造を基本とする。
`Visual Brief` の内容に合わせて不要な項目は削除する。

```text
日本語のグラフィックレコーディングを1枚生成してください。

【目的】
現在の会話、資料、またはGit差分の最終的に重要な内容を、
元の文脈を知らない人でも1枚で理解できるように可視化する。
会話ログやコード全文を載せるのではなく、結論、変更点、関係性、流れ、条件、次の一歩を示す。

【比率】
[16:9 または 4:5]

【スタイル】
[general / business / learning / roadmap / system / live / story / magazine]
白または淡い紙色の背景。太めで読みやすい手描き線。
色は3〜5色程度。子ども向けになりすぎない。
写真風、3D風、リアル人物写真風にはしない。
装飾より情報理解、視線誘導、文字の読みやすさを優先する。

【タイトル】
[短いタイトル]

【最重要メッセージ】
[中央または最上部に大きく置く1文]

【情報ブロック】
1. [見出し]
   - [短い要点]
   - [短い要点]
2. [見出し]
   - [短い要点]
   - [短い要点]
3. [見出し]
   - [短い要点]
   - [短い要点]
[必要なら最大6ブロックまで]

【関係・流れ】
- [A] → [B]: [関係を表す短語]
- [B] → [C]: [関係を表す短語]

【次の一歩】
[短い行動]

【未確定事項】
[ある場合だけ。結論と視覚的に区別する]

【必ず正確に保持する文字列】
- [製品名、固有名詞]
- [日付、数字]
- [専門用語]

【画像内テキストの制約】
- 日本語を基本とする。
- 長文を入れない。
- 短い見出しと短いフレーズへ圧縮する。
- 情報ブロックは3〜6個。
- 小さい文字を大量に配置しない。
- 同じ内容を重複させない。
- 入力にないキャッチコピー、事実、数値、因果関係を作らない。
- 出典URLを小さく詰め込まない。
```

## 差分向け追記

Git差分を可視化する場合は、必要に応じて次を追加する。

```text
【差分の表現】
- ファイル名の羅列ではなく、機能、層、責務でまとめる。
- 「変更前」「主要変更」「変更後・影響」が視線の流れで分かる構成にする。
- 追加、変更、削除を色または記号で区別する。
- 品質の良し悪しは評価しない。
- 推定した影響には「推定」と付ける。
- staged / unstaged / untracked が重要な場合は下部に状態として示す。
```
