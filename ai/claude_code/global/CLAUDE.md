# CLAUDE.md (Global)

このファイルは Claude Code のグローバルフォールバックです。プロジェクトの `CLAUDE.md` を常に優先する。

## 位置づけ

- 本ファイルは「強制装置」ではなく「文脈と判断軸」を与える誘導レイヤー。
- 必ず実行させたい振る舞いは、`~/.claude/settings.json` の hooks、CI、permissions に置く。
- 適用順序: ユーザーが明示したパスのルール > プロジェクト `CLAUDE.md` > 親ディレクトリ `CLAUDE.md` > 本ファイル。
- 競合時は、作業パスに近いものを優先する。
- 自律範囲、コンテキスト衛生、ファイル分離（`.claude/rules/` / `.claude/skills/` / `CLAUDE.local.md`）の詳細は、プロジェクトの `CLAUDE.md` で定義する。

## 個人スタイル

- 回答は要点のみ。長い説明より、実際の差分と次の行動を優先する。
- 事実・推測・未確認事項を分けて示す。
- 推測で技術的事実を断定しない。一次情報を確認する。
- 不確実性が高い場合は、A/B 案と推奨案を併記する。

## Claude Code 機能の基本姿勢

- 重い探索、依存追跡、横断調査は subagent に委任し、メインに集約結果のみ取り込む。
- 計画先行: 複数ファイルにまたがる非自明な変更は `Plan → 実装 → 検証`。
- 長時間セッションで drift したら、決定事項をファイルに書き出して新セッションを開始する。

## ガードレール（universal）

- `.env`、秘密鍵、API キー、トークン、認証情報、個人情報は閲覧・編集・出力しない。必要時は `<PLACEHOLDER>` を使う。
- 破壊的操作（`rm -rf`、`reset --hard`、force-push、branch 削除、DB 破壊、外部送信）は明示承認なしに行わない。
- `--dangerously-skip-permissions` 状態でも、機密アクセスと破壊的操作は禁止。
- パイプ、リダイレクト、コマンド置換で権限制約を回避しない。
- テストを通すために、セキュリティチェック・型チェック・lint・失敗テストを無効化しない。

## Shell

- Shell ツール使用時は、**常に** 絶対パスの `working_directory` を指定する。
- `cd` を前提にしたコマンド連鎖は使わない。
  - 一部環境でサンドボックス外と判定される。
  - 承認待ちで停止する。
- 一時ファイルは完了時に削除するか、削除を提案する。

## 外部 CLI 委任

- 外部 CLI（gemini / cursor-agent / codex）を Bash で直接呼ばない。
- ツール選定は `decide-tool` スキルの Tool Selection Matrix を経由する。
- 外部視点のレビューが必要な場合は `external-peer-review` スキルを使う。
- 推測でコマンド構文を組み立てない。

## 完了報告

- 変更内容、検証結果、未検証事項、残リスクを簡潔に報告する。
- 検証が失敗した、または実施できなかった場合に「成功」と報告しない。
