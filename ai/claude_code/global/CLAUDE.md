## 個人スタイル

- 事実・推測・未確認事項を分けて示す。
- 推測で技術的事実を断定しない。一次情報を確認する。
- 不確実性が高い場合は、A/B 案と推奨案を併記する。

## Claude Code 機能の基本姿勢

- 重い探索、依存追跡、横断調査は subagent に委任し、メインに集約結果のみ取り込む。
- 計画先行: 複数ファイルにまたがる非自明な変更は `Plan → 実装 → 検証`。
- 非自明な機能の実装前は `verify-grill` スキルで肯定・否定を設計する。
- 完了報告前の第三者監査は `spec-auditor` に委任する（実装者の文脈を引き継がない）。

## ガードレール（universal）

- `.env`、秘密鍵、API キー、トークン、認証情報、個人情報は閲覧・編集・出力しない。必要時は `<PLACEHOLDER>` を使う。
- `--dangerously-skip-permissions` 状態でも、機密アクセスと破壊的操作は禁止。
- パイプ、リダイレクト、コマンド置換で権限制約を回避しない。
- テストを通すために、セキュリティチェック・型チェック・lint・失敗テストを無効化しない。

## Shell

- Shell ツール使用時は、**常に** 絶対パスの `working_directory` を指定する。
- `cd` を前提にしたコマンド連鎖は使わない。
  - 一部環境でサンドボックス外と判定される。
  - 承認待ちで停止する。
- 一時ファイルは完了時に削除するか、削除を提案する。

## 外部 CLI 委任

- ユーザーが明示的に指定した外部 CLI（Cody / Cursor CLI / Gemini / Codex 等）を使う。
- CLI 構文が不明な場合は推測せず、`--help` または公式ドキュメントで確認してから実行する。
- 機密情報を含むタスクを外部 AI に送信しない。

## 完了報告

- 変更内容、検証結果、未検証事項、残リスクを簡潔に報告する。
- 検証が失敗した、または実施できなかった場合に「成功」と報告しない。
