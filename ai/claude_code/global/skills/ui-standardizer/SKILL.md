# UI Standardizer

既存のデザインシステム・アクセシビリティ・定数管理を守るための監査スキル。プロジェクトにトークンがなければ、AI 出力でよくある「ありがちな見た目」を避ける指針を適用する。

## デザイン指針（プロジェクトにトークンがない／新規作成時）

- **避ける**: 紫〜青系の線形グラデーション背景、Inter / Roboto / Arial など汎用サンセリフのみの構成、中央寄せヒーロー＋均等カードグリッドのテンプレ配置、影・モーションなしのフラットだけの UI
- **推奨**: タイポグラフィの階層差（サイズ・ウェイト）をはっきりつける、単一のプライマリ色＋濃淡で統一、角丸・シャドウ・控えめなモーションで質感を出す

## いつ使うか

- ユーザーが CSS・Tailwind 設定・コンポーネントのスタイル・レイアウトを編集／追加しているとき
- ユーザーが design system・design tokens・アクセシビリティ・a11y に言及したとき
- スタイルの統一やコントラストの修正を依頼されたとき

## 手順

1. **標準化**: `Grep` で `tailwind.config.*` や `variables.css`（または同等）を特定し、プロジェクトのデザイントークンを把握する。新規・修正コードにマジックナンバーやトークンに反する値を書かないよう監査する。
2. **アクセシビリティ**: ARIA の使い方・コントラスト比・キーボード操作を確認し、WCAG を意識した水準を満たすよう修正する。
3. **定数**: 色・余白・タイポグラフィは CSS 変数や共有定数を使い、直書きを避ける。

## 完了前チェックリスト

- [ ] 新規・変更したスタイルはプロジェクトのトークン／変数を使っており、マジックナンバーになっていない（トークンがなければ上記デザイン指針に沿っている）
- [ ] インタラクティブ要素に適切な ARIA とキーボード操作が備わっている
- [ ] 色・コントラストが可読性の要件を満たしている
