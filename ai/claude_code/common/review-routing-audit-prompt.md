# レビュー導線の競合チェックと修正用プロンプト

## 1. 問題点（コミット前の状態）

グローバルとプロジェクトの両方にレビュー系のスキルやエージェントが存在しているのに、うまく棲み分けできておらず、**競合していて動作が不安定**だった。
そのため、好ましくない状態だった。

## 2. 解決の方法（コミット後の状態）

- **コンテキストベースの自動ルーティング**: 差分取得後、差分量・ファイル数・リスク領域から単一パス vs パイプラインを決める。オーバーライド（「しっかり」→ パイプライン、「軽く」→ 単一パス）は任意。パワーユーザー向けの近年のトレンド（システムが深さを自動調整）に沿う。
- **グローバルスキルは自己完結**: 他プロジェクトのエージェント名・スキル名・ファイルパスを書かない。「ユーザーが差分レビューを依頼したときの手順と出力」だけを書く。
- **プロジェクト側でルーティングを定義**: プレイブックやルールで「差分取得 → ヒューリスティックで判定 → 単一パス or パイプライン」を明文化する。
- **出力形式の統一**: どのルートを通っても同じレポート形式に揃える。
- **設計根拠の明記**: プレイブック等にコンテキストベースにした理由を 1〜2 文で残す。

## 3. 解消済みか

上記を満たしていれば、競合は解消できているとみなしてよい。

---

## 4. 実行用プロンプト

以下をそのまま AI に渡す。プロジェクトにグローバルとプロジェクトの両方にレビュー系スキル/エージェントがある場合、同じ問題を抱えていないかチェックし、抱えていれば上記と同様の最善の解決を進める。（prompt-evolution で最適化済み）

```markdown
# レビュー導線の競合チェックと修正

**Goal**
- (1) グローバルとプロジェクトのレビュー系が、棲み分けできず競合・動作不安定でないかチェックする。
- (2) 問題があれば、コンテキストベース自動ルーティング・グローバル自己完結・出力統一・設計根拠明記の状態に修正する。

**前提**
- グローバル側: ~/.claude/skills/ 等。
- プロジェクト側: .claude/agents/, .cursor/agents/, .codex/, .claude/skills/ 等。

**成功基準**
- チェック: 次の4項目について有無を確認した。
  - (1) トリガー・役割の重複・競合
  - (2) グローバル本文にプロジェクト固有の名前・パス
  - (3) ルートが発話だけで決まる設計
  - (4) 単一パスとパイプラインの出力形式のばらつき
- 修正: 問題があった場合のみ、次の4項目を実施した。
  - (1) グローバルを自己完結に
  - (2) ルーティングを差分・差分量・ファイル数・リスクで決定する設計に
  - (3) 全ルートで同一レポート形式に
  - (4) プレイブック等に根拠を1〜2文で
- 報告: 次を出力した。
  - チェック結果サマリ
  - 修正時: 変更ファイルと要約
  - 不修正時: 理由

**出力形式**
- チェック結果: 4項目の有無を1行ずつ。
- 修正時: 変更したファイル一覧、各ファイルの変更要約（1行）。
- 不修正時: 該当なしとした理由（1〜2文）。
```
