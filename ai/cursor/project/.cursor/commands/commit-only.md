# コミットのみ（現在のブランチ）

## 概要

ローカルの変更をコミットだけ行うシンプルな手順です。プッシュやブランチ戦略は扱わず、**コミットメッセージ規約**に沿ったコミットだけを実行します。

## 前提条件

- 変更済みファイルが存在すること
- コミットメッセージの書式は `.cursor/rules/commit-message-format.mdc` に従うこと

## 実行手順（対話なし）

1. 未コミット差分を確認してメッセージを決める（例: `git diff`, `git diff --cached`）
2. 変更をステージング（`git add -A`）
3. コミット（環境変数または引数でメッセージを渡す）

### A) 一括実行（推奨）

```bash
MSG="<Prefix>: <サマリ（命令形/簡潔に）>" \
git add -A && \
git commit -m "$MSG"
```

例:

```bash
MSG="fix: 不要なデバッグログ出力を削除" \
git add -A && \
git commit -m "$MSG"
```

### B) ステップ実行（確認しながら）

```bash
# 1) 差分を確認
git status
git diff

# 2) ステージング
git add -A

# 3) コミット
git commit -m "<Prefix>: <サマリ（命令形/簡潔に）>"
```

## ノート

- コミットメッセージの生成は必ず差分に基づいて行うこと。
- ブランチ戦略（main 直コミット禁止など）や `git push` は別のコマンド/運用で扱う。
